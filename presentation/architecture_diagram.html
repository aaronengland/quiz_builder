Quiz Builder - Sequence of Events

React 18 + FastAPI + Claude Sonnet 4.5 (AWS Bedrock) + SQLite

1
User Enters a Topic
User types a topic (e.g., "Neural Networks") and clicks "Generate Quiz." The React frontend sends POST /api/quiz/generate. FastAPI validates through GenerateRequest Pydantic model (1-200 chars).
React FastAPI Pydantic
2
Wikipedia API - Context Retrieval
Async GET to /api/rest_v1/page/summary/{topic} via httpx. Returns a 1-3 paragraph extract for factual grounding. 5-second timeout. If it fails, quiz still generates without it.
httpx (async) Wikipedia REST API
3
Prompt Construction - Context Injection
Wikipedia summary is injected into the prompt as reference material, along with the JSON schema and generation rules (difficulty distribution, distractor quality, explanation length). If no Wikipedia context, that block is simply absent.
quiz_generator.py
4
LLM Call #1 - Quiz Generation
Prompt sent to Claude Sonnet 4.5 via bedrock_client.converse(). Generates 5 multiple-choice questions with options A-D, correct answers, and explanations.
AWS Bedrock Claude Sonnet 4.5 Temperature: 1.0 (varied quizzes per topic)
5
Pydantic Validation (with Retry)
Response cleaned (strip code fences), parsed as JSON, validated through GeneratedQuestion model. Enforces: 5 questions, all fields present, correct_answer must be Literal["A","B","C","D"].
Pydantic Retry up to 3x on validation failure
6
LLM Call #2 - Fact-Check Verification
Generated questions + Wikipedia context sent in a new prompt. Claude reviews each answer for factual accuracy and corrects errors. Output re-validated through same GeneratedQuestion model.
AWS Bedrock Claude Sonnet 4.5 Retry up to 3x Fallback: use unverified questions
7
Persist Quiz to Database
Quiz record (topic + timestamp) and 5 Question records (text, options, correct answer, explanation) saved via SQLAlchemy with foreign keys and cascading deletes.
SQLite SQLAlchemy Ephemeral: container lifecycle only
8
Quiz Displayed on the Frontend (Answers Hidden)
Response serialized through QuizOut + QuestionOut. Questions and options A-D are shown, but correct_answer and explanation are excluded. 5 QuestionCard components with radio buttons.
React Pydantic Answers hidden - anti-cheating
9
User Selects Answers and Submits
Answers tracked in React state. Submit button disabled until all 5 answered. Sends POST /api/quiz/{id}/submit with SubmitRequest: a dict mapping question IDs to selected letters.
React FastAPI Pydantic
10
Scoring and Result Persistence
User answers compared to stored correct_answer for each question. Score computed (e.g., 4/5). QuizResult record persisted: quiz ID, score, total, answers as JSON, timestamp.
SQLite SQLAlchemy Persists for container lifetime only
11
Results Displayed on the Frontend
SubmitResponse returns score + all questions via QuestionWithAnswer (correct answers and explanations now revealed). Cached in sessionStorage. Color-coded score badge + per-question review with green/red indicators.
React Pydantic Answers + explanations revealed
